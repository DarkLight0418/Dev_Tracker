# Dev Tracker

개발 과정에서 발생하는 **문제 → 해결 → 검증**의 사고 흐름을 기록하고,
개발자의 판단과 선택을 자산으로 남기는 개발 기록 시스템입니다.

---

## 1. 프로젝트 소개

Dev Tracker는 결과 중심의 포트폴리오가 아니라  
**왜 그렇게 구현했는가를 설명할 수 있는 개발자**를 목표로 합니다.

코드는 바뀌어도,
의사결정과 사고 과정은 남아야 한다는 문제의식에서 출발했습니다.

---

## 2. 왜 이 프로젝트를 만들었는가

개발을 하다 보면 다음과 같은 문제를 반복해서 겪게 됩니다.

- 문제 해결 과정이 이슈, 커밋, 메모 등으로 분산됨
- 시간이 지나면 왜 그런 구현을 했는지 기억나지 않음
- 같은 실수를 반복하거나, 해결 경험을 재사용하기 어려움
- 개발 성장을 설명할 수 있는 정제된 기록이 부족함

Dev Tracker는 이러한 문제를  
**구조화된 기록 시스템**으로 해결하고자 합니다.

---

## 3. 주요 타겟

- 개인 개발자
  - 문제 해결 로그를 체계적으로 남기고 싶은 개발자
  - 학습과 성장을 증명할 기록이 필요한 개발자

- 팀 프로젝트
  - 기술 의사결정 공유
  - 회고 및 온보딩 자료 정리

---

## 4. 핵심 기능

### 4.1 사용자 관리
- 로그인 / 회원가입

### 4.2 개발 로그 관리
- 개발 로그 작성
- 날짜별 로그 조회
- 코드 학습 주제 및 학습 시간 기록

### 4.3 통계 및 요약
- 학습 시간 및 주제 통계
- 월간 개발 요약 제공
- (예정) AI 기반 로그 분석 및 인사이트 제공

---

## 5. 프로젝트 방향성

Dev Tracker는 다음 원칙을 중심으로 설계됩니다.

- 문제 → 구현 → 검증 사이클을 유지
- 결과보다 선택의 이유를 기록
- 실패한 시도도 자산으로 관리
- 빠른 해결보다 이해 가능한 판단을 우선

---

## 6. 문서 구조(점검 예정)

상세한 기록 템플릿과 설계 문서는 `docs/` 디렉토리에서 관리합니다.

```
docs/
├─ problems/
├─ decisions/
├─ retrospectives/
├─ logs/
├─ templates/
```

---

## 7. 기술 스택
- Frontend: React
- Backend: Spring Boot (Java), Python
- Database: MariaDB (MySQL)
- AI 분석: Spring AI (예정), OpenAI LLM (RAG 방식)


---

## 8. 프로젝트 목표
Dev Tracker는
“무엇을 만들었는가”보다
“왜 그렇게 만들었는가를 설명할 수 있는 개발자”를 위한 프로젝트입니다.
